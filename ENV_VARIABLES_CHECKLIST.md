## üêõ Troubleshooting

### "Missing SUPABASE_URL" error:
- Make sure BOTH `SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_URL` are set
- They should have the SAME value

### "Ring tone failed to load":
- This is non-critical, call will work anyway
- Audio files are in `/public` directory

### "No audio energy detected":
- Check microphone permissions in browser
- Try a different browser (Chrome recommended)
- Check WebRTC stats in demo diagnostics
